Description

I designed this series of wireframes for the now defunct registration form builder web application for 8packs when I was working as an intern. The aim of the project is to reduce the time and complexity required to create customised registration forms.

Ideation

8packs is a startup which provides an end-to-end system to race event organisers. The goal is to make registration for race event as easy as a push of a button. The existing business model is to deploy a single backend system for each race event. After much consideration, the founders decided to review the business model into a software as a service (saas) system. Much research were put into this ambitious project as the initial plan was to have a complete overhaul of the existing 8packs system. My role in this project is to draft a wireframe based on the users' stories obtained from existing users' feedback. This wireframe, which is the fourth iteration allows users to set conditional links across different form fields. Unfortunately, the idea was scrapped as there was not enough resources at the time being to build the system.

Specifications

Layout

The layout of the registration form builder can be broken down into three components, the navigation bar, the sidebar, and the main content. The navigation bar shows the progress of the user. This reminds the users that they are currently 80% through with the entire system deployment. The sidebar contains the form field library. Users are able to add new fields onto the registration form by dragging them from the sidebar and dropping them onto the dropzone found on the content section. The content section shows the ordering sequence of the form fields. It also allows users to create and toggle between pages found in the form field.

User flow

Landing page

The user lands on the landing page and choose to create a new registration form.

Create new form

The user can add new form fields by dragging the desired fields onto the content area.

Edit form field attributes

The user can edit the attributes for each form field.

Create conditional links

The user can create conditional relations between the form fields.
